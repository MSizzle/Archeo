  <behavior>
    - A FlowTransition is flagged back:true when EITHER signal fires:
      (a) an agent-step record with agentAction==='back' occurs in the sequence acting from the
          `from` state, and the next navigation returns to a prior state (the executed goBack), OR
      (b) the transition reverses a previously-observed forward transition — an earlier from'→to'
          exists where from'==to and to'==from (a return traversal to an already-visited state).
    - inferFlows interleaves navigation + agent-step records by seq for signal (a); signal (b) is
      computed purely from the ordered forward-transition history.
    - Forward-only fixtures produce zero back-edges (no false positives). A fixture with a genuine
      A→B→A pattern OR a back agent-step yields a transition with back:true.
    - Determinism + first-seen ordering preserved; transition counts still aggregate.
  </behavior>

    later seen, set back:true on that transition. Additionally scan agent-step 'back' records to flag
    the corresponding return transition. Merge both into the transition set (a transition flagged by
    either signal is back:true).
  </action>
