`Observation` (screenshot + interactive-element inventory), key each page by a **SPA-aware state
signature** (AGENT-03), and turn a model reply into a **validated action** (AGENT-06) drawn from
the fixed vocabulary `click/navigate/fill/scroll/back/done` (AGENT-01) — with the **hard
never-click blocklist** (AGENT-07a) applied both before the model sees the inventory and after it
chooses. This is the decision layer promised in D5-01.

Purpose: give the loop (05-03) a clean, tested contract — `captureObservation(page)`,
`computeStateSignature(obs)`, and `decideWithRetry(provider, obs, frontier)` — while keeping the
DOM-touching parts thin and the reasoning parts pure and offline-testable (the scripted provider
from 05-01 drives every test).

Output: `src/agent/observation.ts`, `src/agent/signature.ts`, `src/agent/blocklist.ts`,
`src/agent/decision.ts` and their unit tests. `src/agent/` MAY import `src/spec/templater.ts`

# src/agent/blocklist.ts (a code constant — AGENT-07a)
export const BLOCKLIST_RE: RegExp
  // case-insensitive; matches: logout, log out, sign out, sign off, log off, switch account,
  //   delete account, close account, deactivate, and unsubscribe when 'account' is nearby.
isBlockedElement(el: { text?: string; role?: string; href?: string; id?: string; ariaLabel?: string }): boolean
  // true if BLOCKLIST_RE matches ANY of the element's text/aria/href/id.
annotateBlocklist(inventory: InventoryElement[]): InventoryElement[]
  // returns a copy with el.blocked=true set on matching elements (they are NOT removed — the loop
  //   still records their existence, they are just never offered to the model as actionable).

# src/agent/observation.ts (AGENT-01)
export interface InventoryElement {
  ref: number; tag: string; role?: string; text?: string;      // text truncated to ≤80 chars
  href?: string; inputType?: string; inputName?: string;
  bbox: { x: number; y: number; w: number; h: number }; blocked: boolean;
}
export interface Observation { url: string; title: string; screenshot: ChatContentPart; inventory: InventoryElement[]; }
export const INVENTORY_BROWSER_FN: string
  // the pure browser-side function body (a string passed to page.evaluate) that walks the DOM for
  //   visible interactive elements and returns the raw pre-normalized element list. Kept as a
  //   string constant so its post-processing can be unit-tested without a browser.
normalizeInventory(raw: RawEl[]): InventoryElement[]
  // PURE: assign stable ref indices, truncate text to 80 chars, drop non-visible/zero-box elements,
  //   then annotateBlocklist(...). This is what tests exercise.
captureObservation(page: Page): Promise<Observation>
  // THIN wrapper: page.evaluate(INVENTORY_BROWSER_FN) → normalizeInventory; page.screenshot({type:'jpeg',
  //   quality:60}) → base64 ChatContentPart; page.url()/page.title(). Integration-proven in 05-05.

# src/agent/signature.ts (AGENT-03 — SPA-aware)
export interface SignatureInput {
  route: string;                                  // pathname (templated inside)
  landmarks: { nav: number; main: number; dialog: number; form: number; headings: string[] };
  inventory: InventoryElement[];
}
computeStateSignature(input: SignatureInput): string
  // sha256 hex of `${templatePath(route)}|${landmarkKey(landmarks)}|${elementShapeKey(inventory)}`.
landmarkKey(l): string        // stable: counts + sorted heading texts joined
elementShapeKey(inv): string  // stable: sorted multiset of `${tag}:${role}:${inputType??''}` (NOT text/href — shape only)

# src/agent/decision.ts (D5-01 decision layer + AGENT-06 + AGENT-01)
export const ACTIONS = { CLICK:'click', NAVIGATE:'navigate', FILL:'fill', SCROLL:'scroll', BACK:'back', DONE:'done' } as const
export interface AgentAction { action: Action; targetRef?: number; value?: string; reasoning: string; }
buildObservationPrompt(obs: Observation, frontier: FrontierSummary): ChatMessage[]
  // system rules + a user message: a text summary (route, landmarks, actionable inventory with the
  //   blocked ones excluded, frontier summary) + the screenshot image part + an EMBEDDED machine
  //   envelope (```json { inventory:[refs], frontier:[refs] }```) so the scripted provider can decide.
parseDecision(raw: string, inventory: InventoryElement[]):
  { ok: true; action: AgentAction } | { ok: false; reason: string }
  // strict JSON.parse; action ∈ ACTIONS; if targetRef present it must index a NON-blocked inventory
  //   element (post-decision blocklist recheck — a blocked or out-of-range ref → { ok:false, reason }).
decideWithRetry(provider: Provider, obs: Observation, frontier: FrontierSummary):
  Promise<{ action: AgentAction; source: 'model' | 'fallback' }>
  // call provider.chat(prompt) → parseDecision; on !ok, re-prompt ONCE with the failure reason
  //   appended as feedback; on a second !ok, return { action:{action:'back',reasoning:'fallback: …'},
  //   source:'fallback' } so the loop pops the frontier (AGENT-06 — never crash, never guess).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model reply → browser actions | A model reply is untrusted input; it must be validated against the live DOM before any action executes (AGENT-06). |
| agent → live account (blocklist) | The model must never be able to drive a logout / account-switch / delete-account control (AGENT-07a). |
| target page text → model provider | The inventory text (≤80 chars/element) + screenshot are sent to the provider — the user's explicit BYO-key configuration (D5-01), reiterated here. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-06 | Tampering | a hallucinated / malformed model reply driving an invalid or out-of-range action | mitigate | parseDecision strict-JSON parses, checks the six-action vocabulary, and requires targetRef ∈ current inventory; invalid → re-prompt once → frontier-pop fallback; never crash, never guess (AGENT-06). |
| T-05-07 | Elevation of Privilege | the agent logging itself out / switching or deleting the account | mitigate | annotateBlocklist marks such elements before the model sees them (they are excluded from the actionable list) AND parseDecision rechecks the chosen targetRef against isBlockedElement (defense in depth) — AGENT-07a. |
| T-05-08 | Denial of Service | a provider that always returns garbage looping forever | mitigate | decideWithRetry re-prompts exactly once then returns a deterministic fallback; the loop's step budget (05-03) bounds the run regardless. |
| T-05-09 | Information Disclosure | over-long or sensitive element text shipped to the provider | mitigate-partial | inventory text is truncated to ≤80 chars/element; only actionable non-blocked elements are summarized; the screenshot disclosure is the accepted, user-configured BYO-key posture (T-05-04). |

No package installs; no new outbound surface (src/agent imports src/model + src/spec + playwright
types only). The no-network guard stays green.
