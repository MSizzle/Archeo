<objective>
Build the **autonomous explorer loop** and the **`archeo explore`** CLI. The loop turns the
05-02 primitives into directed coverage: a **coverage graph + prioritized frontier** (AGENT-04),
**loop detection + backtrack** (AGENT-07b), **stop conditions with a recorded stop reason**
(AGENT-05), **synthetic form-fill** (AGENT-02), and a new **agent-step store record** that both
the dashboard and the spec's flows consume. `archeo explore` runs gate-first, reuses the Phase-4
persisted profile, keeps the **floor ON (writes held — non-negotiable; no `--allow-writes` in this
phase)**, and auto-generates the deterministic spec exactly like manual mode.

Purpose: deliver the "no human clicks required; coverage climbs then plateaus" success criterion
safely — the loop can drive a real authenticated app without logging itself out, without spinning
forever, and without any write reaching the server.

Output: `src/agent/{graph,loopDetect,stop,formfill,loop}.ts`, `src/cli/explore.ts`, the
`explore` subcommand wired into `src/cli/index.ts`, the agent-step record type in
`src/types/index.ts` + `src/capture/store.ts`, and their tests. All automated tests run on the
**scripted provider** (no key, no network).
</objective>

export interface StateNode { signature: string; url: string; title: string; firstSeenStep: number; }

  addState(node: StateNode): { isNew: boolean }
  addTransition(from: string, to: string, action: string): void
  addFrontier(items: FrontierItem[]): void            // dedup; never re-add an exercised item

}
