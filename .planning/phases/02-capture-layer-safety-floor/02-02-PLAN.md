<objective>
Refine the floor so it is protocol-aware and its held-write responses are shaped. GraphQL queries/introspection and JSON-RPC reads — which plan 01 conservatively held as POSTs — now correctly pass and are captured as reads, while GraphQL/JSON-RPC mutations stay held. Held writes return a synthetic 2xx shaped from a prior observed redacted response for the same path (D-03), so the page behaves realistically without ever echoing the request payload back.

Purpose: A real user can now browse GraphQL/JSON-RPC apps without their read traffic being needlessly blocked, and held writes look successful to the app. Safety is never reduced — a mutation is still never allowed through.

Output: extended `classifier.ts` (GraphQL/JSON-RPC), `store.ts` (response corpus), `interceptor.ts` (shaped synthetic responses), and extended test suites.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → target server | A GraphQL/JSON-RPC mutation could reach the server if mis-classified as a read |
| held request payload → synthetic response → page | The held request body could leak back into the page if echoed as the synthetic response |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering | classifyRequest GraphQL/JSON-RPC dispatch | mitigate | GraphQL/JSON-RPC body checked BEFORE REST fallthrough; mutations held; JSON-RPC fail-closed (only explicit read prefixes pass). Test asserts mutation held. |
| T-02-08 | Information Disclosure | held-write synthetic response | mitigate | Synthetic body sourced only from store.findSimilarResponse (redacted corpus) or generic fallback; request.postData never reaches route.fulfill body. Test asserts no echo. |
| T-02-08b | Information Disclosure | response corpus contents | mitigate | Corpus stores only the already-redacted responseBody (JSON.stringify of redact* output), so reused shapes carry no secret values (CAP-05 invariant preserved). |

No package installs in this phase — no supply-chain (T-*-SC) checkpoint required.
</threat_model>
