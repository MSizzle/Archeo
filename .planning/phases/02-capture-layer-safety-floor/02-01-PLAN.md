<objective>
Deliver the thinnest real end-to-end capture slice: a human drives the existing headed Chromium, and every target-scoped request is intercepted, classified, redacted in-memory, and appended to an on-disk JSONL capture store — with all mutating REST requests held before they reach the server.

Purpose: Prove the load-bearing path (intercept → classify → act → redact → persist) works against a live account safely. This is the floor the whole project rests on: the same code that protects the account is the code that captures the blocked request.

Output: `src/capture/{classifier,redactor,store,interceptor}.ts`, extended `src/types/index.ts`, capture wiring in `src/cli/{browser,index}.ts`, and four `test/capture/*.test.ts` suites. After this plan a real user gets a redacted capture store with reads captured and writes blocked.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → target server | A mutating request the user (or app JS) issues could change real account data if it reaches the server |
| captured traffic → disk (JSONL store) | Live auth tokens/cookies and arbitrary field values could persist to disk |
| target origin → third-party hosts | Analytics/CDN traffic carrying the user's cookies could be recorded if not scoped out |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | classifyRequest / handleRoute held path | mitigate | All non-read REST methods classified held:true; held path never calls route.fetch; returns synthetic 200. Test asserts route.fetch not called for POST. |
| T-02-02 | Information Disclosure | redactHeaders + store.append ordering | mitigate | AUTH_HEADER_BLOCKLIST strips Authorization/Cookie/bearer values in-memory; store.append only ever receives redact* output (CAP-02/05). Test asserts no auth value in JSONL. |
| T-02-03 | Information Disclosure | redactValue dual gate | mitigate | Key-category + value-shape gate; any non-match → inferType (CAP-03/05 fail-closed). Test: email field → 'string'. |
| T-02-04 | Information Disclosure | isTargetScope route filter | mitigate | context.route URL predicate intercepts target origin + subdomains only; third-party hosts pass untouched and are never written (D-02). Test: isTargetScope('evil.com') false. |
| T-02-05 | Information Disclosure | capture store location | mitigate | Store written under gitignored `.archeo/captures/`; .gitignore updated so the redacted store is never committed. |
| T-02-06 | Denial of Service | route handler | mitigate | Whole handler wrapped in try/catch → route.continue(); a throwing handler never leaves a request pending (Pitfall 2). Oversized/binary bodies recorded as metadata only (Pitfall 5). |

No package installs in this phase (zero new runtime dependencies per RESEARCH § Package Legitimacy Audit) — no supply-chain (T-*-SC) checkpoint required.
</threat_model>
