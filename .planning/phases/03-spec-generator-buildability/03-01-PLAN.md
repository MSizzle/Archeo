<objective>
Build the deterministic **endpoint templater** — the pure heart of the spec generator. It collapses
observed concrete request paths into stable endpoint templates (`/users/123` + `/users/456` →
`/users/{id}`), dedups polling / list-refresh noise, and groups records into `EndpointTemplate`
objects that both the spec generator (03-02) and the live dashboard (03-03) reuse. Per D3-01 this
is fully deterministic — no model, no I/O. Per D3-02 the module is pure (zero deps), so it is
exhaustively unit-testable and reused incrementally by the dashboard.

Purpose: SPEC-01 (endpoints collapse into templates) and SPEC-02 (polling/list-refresh dedup) are
delivered here as tested pure functions, with no wiring, so waves 2 and 3 build on a proven core.

Output: new `src/spec/templater.ts` (pure functions), new `src/types/spec.ts` (`EndpointTemplate`),
one optional field added to `CaptureRecord` in `src/types/index.ts`, and a thorough test suite.
</objective>

# EndpointTemplate (src/types/spec.ts):
EndpointTemplate {
  method: string; pathTemplate: string; protocol: Protocol; operationType: OperationType;
  held: boolean; observationCount: number; examplePaths: string[];   // <= 3 distinct concrete pathnames
  statusCodes: number[];            // distinct observed response status codes, ascending
  requestBodyShape: unknown | null;  // latest redacted request body shape in the group
  responseBodyShape: unknown | null; // latest redacted response body shape in the group
  polling: boolean;                  // true when a concrete URL in this group repeated >= 3 times
  operationName?: string;            // GraphQL only
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| capture store record → EndpointTemplate | Templating could collapse two genuinely distinct routes into one, corrupting the spec's endpoint set |
| redacted record shapes → EndpointTemplate.*BodyShape | Body shapes carried into templates must remain the already-redacted shapes (no re-derivation of values) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | templatePathSegment over-templating | mitigate | Conservative, priority-ordered detectors; short alpha slugs are NEVER templated (D3-02). Tests assert /api/users and /api/orders stay distinct. Fail-safe: when unsure, do not template. |
| T-03-02 | Information Disclosure | EndpointTemplate.requestBodyShape / responseBodyShape | mitigate | Templater is pure and only copies already-redacted CaptureRecord bodies (CAP-05 upheld upstream); it never reads raw traffic and never un-redacts. No I/O, so nothing is persisted here. |
| T-03-03 | Tampering | polling mis-count collapsing legitimately distinct calls | accept | Polling only sets a boolean flag; it never drops or merges templates beyond the (method,templatedPath,protocol) key. Worst case is a benign polling:true annotation. |

No package installs in this plan — no supply-chain checkpoint required. No network surface added.
</threat_model>
