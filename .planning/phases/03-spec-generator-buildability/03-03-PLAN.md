<objective>
Stand up the first slice of the live dashboard (D3-05, D13): a localhost-only web app the CLI serves
while the user browses, streaming discovery counts and freshly-discovered endpoints over SSE so
endpoints appear within seconds of the first page load. It reuses the pure templater (03-01) for
incremental aggregation and a new `CaptureStore.onRecord` observer as its event source. Because this
introduces the project's first inbound HTTP server, it also evolves the GATE-03 static guard so an
inbound loopback server is permitted under `src/dashboard/` while every outbound HTTP surface stays
forbidden everywhere.

Purpose: DASH-01 (localhost SSE dashboard), DASH-02 (counts climb in real time as discovery, not a
completion bar), and DASH-03 (time-to-first-magic — endpoints appear promptly, one event per record).

Output: new `src/dashboard/server.ts` + `src/dashboard/page.ts`, a `CaptureStore.onRecord` hook, CLI
wiring with `--no-dashboard` + `--dashboard-port`, the evolved GATE-03 test, and a server smoke test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| inbound HTTP client → dashboard server | A non-loopback bind would expose live-session discovery data to the local network |
| store record → SSE event payload | SSE events must carry only already-redacted aggregates/shapes, never raw values |
| dashboard module → outbound network | GATE-03: the dashboard must never become a phone-home surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Spoofing / Elevation of Privilege | dashboard server bind address | mitigate | server.listen binds '127.0.0.1' explicitly (loopback only) — never 0.0.0.0. A structural GATE-03 test asserts the 127.0.0.1 host literal in listen(. No remote host can reach it. |
| T-03-10 | Information Disclosure | SSE event payloads | mitigate | Events are built from already-redacted CaptureRecords (CAP-05 upheld upstream) and templater output (type names / path templates); no raw value is ever streamed. Loopback-only bind limits the audience to the local user. |
| T-03-11 | Tampering (phone-home) | src/dashboard/ outbound surface | mitigate | Evolved GATE-03 guard: node:http allowed ONLY under src/dashboard/ and ONLY for serving; http.request/http.get are dashboard-scoped forbidden tokens; node:https/axios/undici/got/bare-fetch forbidden everywhere. Server makes zero outbound calls. |
| T-03-12 | Denial of Service | observer/SSE error crashing capture | mitigate | onRecord callbacks run in try/catch (Task 1) and the server's per-record push is wrapped too; a dead SSE client is removed on 'close'; a dashboard failure cannot crash the capture session or block exit. |

No package installs in this plan — no supply-chain checkpoint required. The only new HTTP surface is an inbound loopback server, explicitly scoped and structurally guarded.
</threat_model>
