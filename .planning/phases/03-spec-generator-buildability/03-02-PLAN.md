<objective>
Turn the redacted capture store into a machine-readable JSON build spec, and make it easy to get:
a deterministic `generateSpec(sessionDir)` (D3-01/D3-04) that emits data models, endpoints (held
mutations flagged), UI flows, heuristic business-logic rules, and a mandatory coverage block; a
`src/capture/navigation.ts` main-frame navigation tracker (D3-03) so flows can be inferred; an
`archeo spec [captureDir]` subcommand (gate-free, deterministic, testable); and auto-generation on
graceful browser close so a normal session ends with a spec on disk.

Purpose: SPEC-03..07 are delivered end-to-end from manually captured traffic — the value the whole
project hinges on — with a primary path (`archeo spec`) that is fully unit-testable.

Output: new `navigation.ts` + `generator.ts`, `ArcheoSpec` types, the `spec` subcommand, the browser
auto-gen wiring, and thorough tests. The redaction-fail-closed invariant is untouched: the generator
consumes ONLY already-redacted records.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| navigation URL → capture store | A navigated URL could carry an auth token in the query string |
| redacted store record → ArcheoSpec (on disk) | The spec must never contain a value not already redacted; it is written to disk and later handed to a builder agent |
| graceful-close auto-gen → process exit | Spec generation on close must not deadlock or block the process from exiting cleanly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04 | Information Disclosure | navigation record URL | mitigate | attachNavigationTracker stores redactUrl(frame.url()) — sensitive query params masked before append (CR-02 reused). requestHeaders {} / requestBody null carry no secrets. |
| T-03-05 | Information Disclosure | ArcheoSpec on disk | mitigate | The generator reads ONLY already-redacted CaptureRecords (CAP-05 upheld upstream); dataModel field values are inferred type names, never raw values. Test asserts no planted secret survives into the spec JSON. archeo-spec.json lands under .archeo/ (gitignored). |
| T-03-06 | Denial of Service | graceful-close auto-gen hang | mitigate | gracefulShutdown is idempotent and awaits only the store flush Promise (resolves on 'finish' OR 'error'); writeSpec is wrapped in try/catch so any failure prints a warning and still calls process.exit(0). No timers, no unbounded waits. |
| T-03-07 | Tampering | spec subcommand bypassing the gate | accept | `archeo spec` intentionally skips the authorization gate (no browsing / no target contact happens — it only reads a local store). The `<url>` browsing path keeps its gate-first ordering unchanged (T-01-09 regression-guarded). |
| T-03-08 | Tampering | navigation records corrupting floor/corpus | mitigate | Navigation records are held:false and non-'request-response', so heldWriteCount and responseCorpus (findSimilarResponse) are provably unaffected; full existing suite is a regression gate. |

No package installs in this plan — no supply-chain checkpoint required. No outbound network surface added (generator uses node:fs only).
</threat_model>
