<objective>
Build the authorization gate — the legal and ethical heart of Archeo. Running `archeo <url>` must print the attestation BEFORE anything else happens, then require an affirmative act (a single y/N keypress defaulting to No) or the `--i-have-authorization` flag, and must phone home to nobody.

This plan is TDD: the gate's behavior is fully specifiable as input→output, so tests are authored first (RED), then `src/cli/gate.ts` is implemented to green (GREEN). The gate is deliberately hand-rolled from Node built-ins (readline + process) so that zero npm packages touch the gate path — that is what makes the GATE-03 "no telemetry" guarantee structural rather than aspirational.

Purpose: Satisfy GATE-01 (attest before browser), GATE-02 (flag satisfies gate but attestation still prints), GATE-03 (no phone-home), and D-05 (non-TTY without flag errors out — no silent bypass).
Output: `src/cli/gate.ts` with a green gate test suite and a static no-network guard test. No browser yet (Plan 03 wires it).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user terminal → process | The y/N keypress is the affirmative authorization act; the TTY state must survive interruption |
| process → outside world | GATE-03 forbids any outbound call beyond the user's eventual target; the gate path must touch the network never |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03 | Spoofing / Repudiation | runAuthorizationGate attestation print | mitigate | Attestation is written as the first statement before any branch, including the `--i-have-authorization` path. A failing test asserts ATTESTATION_TEXT prints even when the flag is set. **Block-worthy:** any code path that satisfies the gate without the attestation printing is a high-severity defect. |
| T-01-04 | Elevation of Privilege | non-TTY silent bypass (D-05) | mitigate | `decideGateMode` returns 'error' for `!hasFlag && !isTTY`; runAuthorizationGate exits 1 with a clear stderr message. No piped/CI run can pass the gate without the explicit flag. Unit-tested. |
| T-01-05 | Information Disclosure | telemetry / phone-home (GATE-03) | mitigate | Gate imports only `node:readline` / `node:process`; test/security/no-network.test.ts statically scans src/ for forbidden network tokens and fails the build on any match. |
| T-01-06 | Denial of Service | TTY left in raw mode after Ctrl+C (Pitfall 3) | mitigate | A `process.once('SIGINT', restore)` handler calls `setRawMode(false)` before exit; the handler is removed after the keypress. A non-TTY never calls setRawMode (Pitfall 2 guard). |
</threat_model>
