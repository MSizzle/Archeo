<objective>
Deliver the authentication handoff's foundation: **persistent per-hostname profiles** and the
**`archeo login`** manual-login handoff, built so that login mode can never touch the capture
store (D4-01). This wave refactors `src/cli/browser.ts` from `chromium.launch()` +
`browser.newContext()` to `chromium.launchPersistentContext(userDataDir)` for **both** modes
(D4-03), adds a pure profile-dir resolver (`src/cli/profile.ts`), and adds an isolated login
browser module (`src/cli/login.ts`) plus the `archeo login <url>` subcommand with the terminal
ready prompt (D4-04).

Purpose: AUTH-01 (the user logs in manually, including MFA, and signals ready; Archeo never
handles credentials) and AUTH-02 (the authenticated context is persisted and subsequent runs
explore from there) are delivered as tested code, with the D4-01 no-capture-during-login
guarantee enforced *structurally* (the login module does not import any capture code).

Output: new `src/cli/profile.ts` (pure), new `src/cli/login.ts` (login-mode browser, capture-
free), a refactored `src/cli/browser.ts` (capture mode on a persistent context), `src/cli/
index.ts` wiring the `login` subcommand and threading the profile dir into the capture command,
and tests: pure profile-resolution unit tests, the D4-01 import-isolation structural test, and
CLI spawn tests for the login command's pre-browser guards.
</objective>
