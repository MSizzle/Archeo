subagent NOTHING but the repo, tell it to ignore `.planning/` and act like a stranger, and have
it produce a spec from the **README quickstart alone** — the key-free manual path against a local

LICENSE/NOTICE intact). If the quickstart doesn't work as written, that is a BLOCKING finding —
fix it minimally and re-verify before closing. Then close Phase 7.

REQUIREMENTS). No `src/` or `test/` change unless a blocking finding forces a minimal doc fix.

| documented surface ↔ shipped surface | A flag in the docs that isn't in the code (or vice versa) erodes trust; the audit table closes it. |

| Threat ID | Category | Component | Disposition | Mitigation |

| T-07-06 | Repudiation (docs claim a path that doesn't work) | README quickstart | mitigate | Fresh-eyes subagent runs the quickstart end-to-end from README alone and must produce a spec; failure is blocking and fixed before close. |
| T-07-07 | Information Disclosure | examples/ re-audited | mitigate | Secret-clean grep re-run over examples/ during the audit (Task 3). |
| T-07-08 | Tampering (silent doc/code drift) | README+CONTRIBUTING vs src/cli | mitigate | Doc-vs-code audit table maps every documented command/flag to a source registration. |

No package installs; no new outbound surface; the no-network guard is re-run as part of the suite.
