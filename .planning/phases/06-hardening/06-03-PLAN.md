<objective>
Keep a long unattended run alive through the messy reality of a live app, and keep the operator's
screen calm while doing it. The load-bearing fix — **05-05 finding #1** — is that the loop today
only survives SPAs: `page.evaluate` throws `Execution context was destroyed` the instant a real
cross-document navigation fires, killing the run. `observeWithRecovery` catches that, waits for
the page to settle, and re-observes with bounded retries, so the loop now survives **real
full-page navigations**. Around it, `src/agent/recovery.ts` classifies errors and drives the
recoverable ones (nav timeout, model error, action failure, dead-end) back onto the frontier
without a stack trace, logging one calm line each to a **rotating in-memory issue log** + a muted
SSE `error` event. Run-halting classes (browser gone, target unreachable ×3, budget mid-action,
auth expiry — 06-04) surface **loudly**: a prominent dashboard banner, a run-state change, and a
single terminal line. The terminal stays quiet for everything recoverable (DASH-08).

Output: `src/agent/recovery.ts`; recovery wired into `src/agent/loop.ts`; the quiet-vs-loud error
surface in `src/dashboard/{server,page}.ts` + `src/cli/explore.ts`. All automated tests run on the
scripted provider with a fake page that simulates a context-destroyed throw and a flaky endpoint.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| observation ↔ real cross-document navigation | page.evaluate races with a full navigation; observeWithRecovery must catch the context-destroyed throw, settle, and re-observe — otherwise the loop only ever works on SPAs (the 05-05 demo-grade limitation). |
| recoverable error ↔ operator attention | A recoverable error must never spam stderr or raise a banner; only genuinely run-halting conditions may interrupt the operator (DASH-08). |
| issue-log strings ↔ dashboard DOM | Error messages (which may contain target-derived text) are rendered textContent-only, never innerHTML — no injection from a hostile page's error text. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-06-07 | Denial of Service (run death on real navigation) | captureObservation vs cross-document nav | mitigate | observeWithRecovery catches CONTEXT_DESTROYED, settles, re-observes (bounded retries); 06-06 proves it live against a REAL cross-document-navigation target. |
| T-06-08 | Denial of Service (run death on transient failures) | nav/model/action errors | mitigate | Per-class recovery (retry/backoff/re-observe/backtrack) keeps the run alive; only halting classes stop it. |
| T-06-09 | Information Disclosure / Tampering | error text rendered in the dashboard | mitigate | Issue messages rendered textContent-only; the dashboard stays playwright-free and inbound-only (no-network guard unchanged). |
| T-06-10 | Repudiation | silent failures hidden from the operator | mitigate | Every recoverable error is one line to the rotating issue log + a muted counter; halting errors are loud — nothing fails invisibly. |

No package installs; no new outbound surface. The no-network guard stays green.
</threat_model>
