<objective>
Make a long run survivable across a **mid-run login expiry** and make **re-runs incremental and
diff-aware**. When the session cookie dies mid-run (a 401/403 spike or a redirect to a login
page), the loop pauses cleanly, saves the coverage graph + frontier to `resume.json`, keeps the
browser open on the login page, and asks the human to log back in and press Enter — and **while
paused the interceptor passes every request through UNRECORDED**, the exact D4-01 trust model
that `archeo login` already relies on (the human is driving; credential POSTs must pass; nothing
is captured). On resume it verifies auth is back, reloads the graph/frontier, and continues.
Separately, `archeo explore --resume` seeds a fresh run from the latest prior session for the same
hostname (the graph is now persisted at every stop, not only on pause), and `archeo diff` +
auto-diff-at-explore-end produce a deterministic drift report — new/removed endpoints, removed
pages, changed field shapes, held-status flips — to `drift-report.json`, a stdout table, and a
dashboard event.

Output: `src/agent/{authWatch,resume}.ts`; `src/spec/drift.ts`; a pause flag in
`src/capture/interceptor.ts`; graph seed/serialize helpers; pause/resume + persistence wired into
`src/agent/loop.ts` + `src/cli/explore.ts`; `--resume` + the `diff` subcommand in
`src/cli/index.ts`; a drift SSE event in the dashboard. Unit tests cover the detector, the pause
flag routing, resume reload, and the pure diff; the full pause→relogin→resume is proven live in
06-06.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pause-mode pass-through ↔ capture store | While paused for re-auth, EVERY request (including credential POSTs) passes through UNRECORDED — the human is driving, same trust model as archeo login (D4-01). The pause path must never append a record or populate the corpus, or a credential could land on disk. |
| resume.json ↔ session integrity | resume.json carries coverage-graph structure + frontier (state signatures, URLs, routes) — no credentials, no bodies. A corrupt file fails to null (cold start), never crashes the run. |
| drift reports ↔ stale-spec trust | The diff must be deterministic with zero false positives on an unchanged surface, or a consumer will chase phantom changes; identical specs must produce an empty report. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|------------|
| T-06-11 | Information Disclosure | credentials captured during a paused re-login | mitigate | The interceptor pause flag passes ALL requests through with route.continue and ZERO store.append — no classify, no redact, no record (D4-01). Unit-proven (paused POST+GET → zero records). |
| T-06-12 | Denial of Service | a mid-run 401 spike wasting the whole run | mitigate | AuthWatch pauses + persists resume.json; the run resumes from the saved frontier with a monotonic state count (live-proven in 06-06). |
| T-06-13 | Tampering (stale-spec trust) | false-positive drift reports | mitigate | diffSpecs is pure + deterministic + sorted; identical specs → empty report (tested), so an unchanged surface never reports drift. |
| T-06-14 | Information Disclosure | resume.json leaking secrets | mitigate | resume.json carries only graph structure (signatures, URLs, routes) + frontier — no headers, bodies, or credentials; corrupt → null (cold start). |

No package installs; no new outbound surface. The no-network guard stays green (resume/drift import only node:fs + siblings).
</threat_model>
